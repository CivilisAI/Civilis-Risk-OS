## Review Goal

The top-level README stays product-first.

The evaluation documents in `docs/` exist so Build X reviewers can quickly
check:

- compliance with the public-repo requirement
- the canonical proof path
- the explicit boundaries of what is and is not claimed
- the project’s fit as a reusable Skill on X Layer
