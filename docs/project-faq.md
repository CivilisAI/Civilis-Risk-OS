# Project FAQ

## Is this just a Civilis page feature?

No.

Civilis Intel Market is the live reference integration, but the submission also
includes:

- a second portability adapter
- a machine-readable contract
- AI-readable skills
- an integration checklist

The intended claim is a reusable protection primitive, not a single page.

## Is this a generalized insurance protocol?

No.

It is a narrow protection layer for agent commerce.

It proves:

- quote
- challengeable settlement
- claim
- evaluator resolution
- later repricing

It does not claim:

- generalized insurance reserve
- decentralized arbitration
- partial refunds

## Why is there only one live mainnet settlement loop?

Because the submission is optimized for strength and honesty.

One clean live canonical loop is stronger than multiple shallow, weakly proven
integrations.

The second adapter exists to prove portability, not to fake a second live loop.

## Why isn’t generic buyer wallet-signature proof claimed?

Because the current official Agentic Wallet CLI does not expose a public generic
`message-sign` command for arbitrary claim payloads.

The repo distinguishes between:

- capability that exists
- proof that has been captured publicly

That boundary is deliberate and honest.

## Why does this belong in Skills Arena?

Because the repo is packaged as a reusable skill:

- skill pack
- OpenAPI contract
- schemas
- examples
- replay guidance
- integration guidance

It is intended to be learned and reused by other agents and other apps.
