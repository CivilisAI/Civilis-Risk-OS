# Submission Claims Reference

## Strong Claims We Can Make

- Civilis Risk OS is a reusable protection Skill for agent commerce on X Layer.
- It has one live reference integration through Civilis Intel Market.
- It has one secondary portability adapter proving the pattern can normalize a
  different commerce surface.
- It uses the official Onchain OS capability stack, especially Agentic Wallet
  and x402 payment.
- It proves `quote -> challengeable buy -> claim -> evaluator resolution ->
  later repricing` on X Layer mainnet.

## Claims We Should Not Make

- generic buyer wallet-signature captured proof is complete
- decentralized arbitration is implemented
- partial refunds are implemented
- the second adapter is a second live mainnet settlement loop
- the project is a generalized insurance reserve

## Reason The Buyer Wallet-Signature Boundary Still Exists

The current official Agentic Wallet CLI exposes wallet and payment flows, but it
does not expose a public generic `message-sign` command for arbitrary claim
payloads.

That means the boundary is tooling exposure, not a contradiction in the Risk OS
architecture.
